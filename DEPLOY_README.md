# Sunwave CRM Dashboard — Deploy & Maintenance

A standalone CRM & Productivity dashboard at:
**https://providentanalytics.github.io/Sunwave/crm/**

Lives in a clean subfolder (`crm/`) — does NOT touch the main Sunwave dashboard at `/`.

---

## File Map

| File | Purpose |
|---|---|
| `dashboard_template_crm.html` | **Source of truth** — UI template with `/*INJECT_*/` placeholders. Edit this when you want to change the look or add features. |
| `build_crm.py` | Builder — reads `report_data.json`, processes data, injects into template, writes `crm/index.html`. |
| `crm/index.html` | **Generated output** — never edit by hand. Overwritten by every CI run. |
| `fetch_excel.py` | Pulls `MASTER_Sunwave_New_PowerQuerry.xlsx` from SharePoint via Microsoft Graph API (client_credentials). Generates `report_data.json`. |
| `.github/workflows/refresh-dashboard.yml` | Hourly cron + manual trigger. Runs the full pipeline and pushes updates. |

---

## Architecture

```
SharePoint xlsx → fetch_excel.py → report_data.json
                                          ↓
                     build_crm.py + dashboard_template_crm.html → crm/index.html
                                          ↓
                                  GitHub Pages serves it
```

- `AZURE_CLIENT_ID`     — Azure AD app registration (UserPermissionManagement)
- `AZURE_TENANT_ID`     — `063ab74f-56d1-429a-b96d-a24a572025de`
- `AZURE_CLIENT_SECRET` — App secret value (not Secret ID)

The Azure AD app needs **Application permissions** (admin-consented) on Microsoft Graph:
- `Files.Read.All`
- `Sites.Read.All`

## Six Tabs

1. **Overview** — KPIs (admit rate, in-pipeline, lost/abandoned), activity leaderboard with 🥇🥈🥉 ranks + admits credited (multi-touch), recent admissions with closer column, stuck opportunities (>7d no human activity, amber @14d, red @30d), open CRM tasks
2. **Pipeline & Funnel** — stage distribution, outcome doughnut, lost/abandoned reason tables, time-to-admit histogram with avg + median
3. **User Productivity** — sortable table: total / calls / notes / email-sms / tasks / meetings / opps / days active / vs. peer median bar; stacked-bar activity-mix chart for top 12
4. **User Effectiveness** — admit rate (multi-touch), median first-response time (hrs), peer-benchmark "faster N% / slower N%" badges
5. **Opportunity Flow** — searchable opp list; click any row to expand a chronological timeline (date / type / status / by / subject / note) of every event
6. **Activity Trends** — stacked daily activity by type, hour×day-of-week heatmap, top-6-users daily time series

## Filters (every tab)

- **Period**: `7D / 30D / 90D / YTD / ALL` — default `30D`
- **Outcome**: dropdown — Active / Scheduled / Admitted / AdmittedLegacy / Lost / Abandoned / All
- **User**: multi-select (sorted by activity count) — Ctrl/Cmd-click to pick multiple
- **View**: `Human Only` (default — excludes auto Workflow/Wave events) or `+ Workflow`

---

## Business Logic

- **Admit credit**: multi-touch. Every user who logged a human activity on an admitted opp gets +1.
- **Stuck opportunity**: outcome ∈ {Active, Scheduled} AND opp created >7d ago AND no human activity in last 7d.
- **Response time**: hours from `created_on` of an opp to that user's first activity on it.
- **"Closer"** on Recent Admissions: most-recent human-activity creator on that opp (regardless of period).
- **Workflow events** (`type = 'Workflow'` or `'Wave'`) are auto-generated by Sunwave. Excluded from "Human Only" view (default).

## Pre-Push Checklist

`build_crm.py` automatically validates:
- ✅ All `/*INJECT_*/` placeholders are filled
- ✅ Brace/paren balance check (best-effort)
- ✅ JSON serialization safety (escapes `</` inside string fields)

Manual checks before any template edit:
1. JS parses cleanly: `node -e "var html=require('fs').readFileSync('crm/index.html','utf8'); var m=html.match(/<script>([\s\S]*)<\/script>/); new Function(m[1]); console.log('OK');"`
2. No credentials embedded in output: `grep -E "ghp_|AZURE_CLIENT_SECRET|client_secret" crm/index.html` should be empty
3. Mobile viewport: confirm `<meta name="viewport" content="width=device-width, initial-scale=1.0">` present

## Common Pitfalls

- **Empty CRM Task tab**: the local `report_data.json` may not yet include the `CRM Task` sheet. Run `python fetch_excel.py` first to refresh from SharePoint.
- **403 from Graph**: AZURE app is missing **Application** (not Delegated) permissions on Files.Read.All / Sites.Read.All, or admin consent wasn't granted.
- **Pages 404**: GitHub Pages may need re-enabling — Settings → Pages → Source: "Deploy from a branch" → main / `(root)`.
- **Charts not rendering**: Chart.js CDN may be blocked by corporate firewall. Try a different network.

---

## Caveats

- Activity dates older than 90 days are still queried but the default 30D period filters them out — switch to `ALL` to see everything.
- `User Effectiveness` tab requires at least 1 opp touched by a user to show that user. Users with no opportunity activity in the period are hidden.
- Multi-touch admit credit means a single admission can credit multiple users.
- Time-to-admit only includes opps where both `created_on` and `admission_date` are valid.
